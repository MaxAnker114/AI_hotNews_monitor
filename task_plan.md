# AI Hotnews Detect - 任务计划

## 目标
构建一个可用的网页版热点识别系统（先完成 Web），使用 OpenRouter 接入 AI 进行热点识别与解释；完成后再开发 Agent Skills。

## 默认需求假设（待用户确认）
1. 用户可粘贴多条新闻文本（或标题+摘要）
2. 系统调用 OpenRouter 对输入内容进行：热点打分、主题聚类、原因解释
3. 前端展示：热点榜单、主题卡片、可视化趋势（简化版）
4. 支持配置 OpenRouter API Key 与模型名称
5. 提供基础测试（单元 + API 路由）

## 阶段
- [x] Phase 1: 需求假设确认与方案冻结
- [x] Phase 2: Web 项目初始化与依赖安装
- [x] Phase 3: OpenRouter API 接入与后端实现
- [x] Phase 4: 独特化 UI 实现与交互联调
- [x] Phase 5: 测试编写与执行
- [x] Phase 6: Agent Skills 开发
- [ ] Phase 7: 验收与交付

## 风险与应对
- OpenRouter 调用失败：增加错误提示与重试建议
- 输入内容质量不足：增加最小样本校验
- 空工作区无现成需求文档：采用默认需求假设并请求人工确认
