# AI 热点识别项目需求说明

## 文档元信息
- 创建日期：2026-04-27
- 状态：V1.0（已确认默认需求）
- 目标优先级：先完成 Web 版可用功能，再开发 Agent Skills

## 项目目标
构建一个可落地的 AI 热点识别系统，用户可输入新闻文本，系统通过 OpenRouter 调用大模型完成热点识别、打分和解释。

## 核心需求
1. 提供 Web 页面，支持多条新闻文本输入（每行一条）。
2. 服务端通过 OpenRouter API 完成 AI 分析。
3. 输出结构至少包含：
   - 全局总结
   - 热点主题列表（含热度分数）
   - 每个热点的解释
   - 关联新闻标题
   - 后续观察建议
4. 支持环境变量配置：
   - OPENROUTER_API_KEY
   - OPENROUTER_MODEL
5. 前端视觉必须具备明显个性，不采用模板化默认风格。
6. Web 功能验证通过后，再进入 Agent Skills 开发。

## 非功能要求
1. 错误提示清晰（如缺少 API Key、输入样本不足、上游请求失败）。
2. 接口调用不暴露密钥到浏览器。
3. 具备基础可测试性（解析逻辑与规范化逻辑应有单元测试）。

## 分阶段交付
1. 需求与方案文档沉淀
2. Web 开发与联调
3. 自动化测试（lint/test/build）
4. Agent Skills 开发
5. 用户验收

## 人工确认点
1. 需求冻结确认（当前已按默认 MVP 冻结）
2. Web 功能演示后确认
3. Agent Skills 设计确认
4. 最终验收确认
