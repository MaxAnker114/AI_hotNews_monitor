# AI 热点识别项目验收说明

## 文档元信息
- 创建日期：2026-04-28
- 版本：V1.0
- 适用范围：Web 版 + API + 基础测试

## 验收范围
- Web 监控看板（热点雷达 / 监控词 / 搜索）
- 多源抓取（HackerNews / Reddit / GitHub）
- AI 相关性判定（OpenRouter）
- 监控词 CRUD 与手动检查
- 数据持久化（.data/hot-monitor.json）
- 基础测试（lint / unit test / build）

## 环境准备
- Node.js 18+（建议 LTS）
- 在 web/.env.local 设置：
  - OPENROUTER_API_KEY
  - OPENROUTER_MODEL=deepseek/deepseek-v3.2

## 启动方式
1. 进入 web 目录
2. 执行 `npm run dev`
3. 打开 http://localhost:3000

## 验收步骤
1. 页面展示
   - 首页可正常加载，看板标题与三大 Tab 可见
   - 默认无数据时提示“暂无热点，先添加监控词并手动检查”

2. 监控词管理
   - 在“监控词”Tab 添加新监控词
   - 新增后列表出现并默认启用
   - 点击“已启用/已暂停”可切换状态
   - 删除后列表移除

3. 手动检查热点
   - 点击“手动检查热点”按钮
   - 若有启用监控词，提示检查完成并刷新热点
   - 热点列表出现卡片，包含标题、来源、相关度、摘要、AI 理由

4. 热点筛选
   - 点击来源筛选按钮（HackerNews/Reddit/GitHub）
   - 列表按来源过滤展示

5. 手动搜索
   - 在“搜索”Tab 输入关键词并选择来源
   - 点击“开始搜索”后显示搜索结果

6. 通知计数
   - 触发手动检查后，“未读通知”计数上升

7. API 验证（可选）
   - POST /api/check-hotspots 返回 { message, created }
   - GET /api/hotspots 返回分页数据与统计
   - POST /api/hotspots/search 返回 results 数组

8. 测试与构建
   - `npm run lint` 无错误
   - `npm run test` 全部通过
   - `npm run build` 成功完成
