# AI 热点识别项目技术方案

## 文档元信息
- 创建日期：2026-04-27
- 状态：V1.0
- 技术基线：Next.js App Router + OpenRouter

## 方案摘要
系统采用前后端一体化的 Next.js 方案：
- 前端页面负责数据输入与结果展示
- Route Handler 作为后端 API 代理并调用 OpenRouter
- 通过环境变量管理密钥，避免泄漏

## 技术栈
- 前端：Next.js 16 + React 19 + Tailwind CSS
- 后端：Next.js Route Handler（App Router）
- AI：OpenRouter Chat Completions API
- 测试：Vitest

## API 对接策略（基于 MCP 最新文档）
参考来源：
- OpenRouter（/websites/openrouter_ai）
- Next.js（/vercel/next.js/v16.2.2）

关键实现约束：
1. OpenRouter 统一入口：`https://openrouter.ai/api/v1` 或 `https://openrouter.ai/api/v1/chat/completions`。
2. 鉴权：`Authorization: Bearer <OPENROUTER_API_KEY>`。
3. 推荐应用标识头：
   - `HTTP-Referer`
   - `X-OpenRouter-Title`
4. 结构化输出建议使用 `response_format`（json_object 或 json_schema）。
5. Next.js Route Handler 使用 `Response.json()`/`NextResponse.json()` 返回 JSON。
6. 环境变量在服务端通过 `process.env` 读取。

## 模块划分
1. 页面层：输入新闻文本、触发分析、展示结果。
2. 业务层：输入解析、JSON 抽取、输出规范化。
3. 接口层：Route Handler 调 OpenRouter 并做错误处理。
4. 测试层：覆盖输入解析和输出规范化。

## 详细流程
1. 用户提交原始新闻文本。
2. 后端将文本解析为结构化新闻条目。
3. 校验最小样本数量（默认 >= 3）。
4. 调用 OpenRouter Chat Completions。
5. 抽取并解析模型 JSON。
6. 归一化结果并返回前端展示。

## 质量与安全
1. API Key 仅保留在服务端环境变量。
2. 对上游错误码和返回异常进行显式报错。
3. 对模型返回做容错抽取，避免因格式噪声导致崩溃。
